# Fix Batch (P0-1 / P0-2 / P1-1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the broken build command, add staged timeout + context to backend HTTP forwarding, and raise the stream buffer limit while reporting (no longer swallowing) scanner errors.

**Architecture:** Three independent, atomic changes. P0-1 is docs-only. P0-2 adds a reused `*http.Client` with a custom `Transport` (connect + response-header timeouts, body read unbounded) to `proxy.Handler` and propagates `r.Context()`. P1-1 raises the `bufio.Scanner` line limit in the handler and emits an `error` event when `scanner.Err()` is non-nil in `convert`.

**Tech Stack:** Go 1.25, standard library (`net/http`, `net`, `bufio`, `time`), `httptest` for tests. All tests run under `go test -race`.

**Branch:** `fix/p0-p1-batch` (already created; spec committed at `docs/superpowers/specs/2026-06-20-fix-batch-design.md`).

```go
func TestHandler_BackendTimeout(t *testing.T) {
	// Backend that hangs without ever sending response headers.
	release := make(chan struct{})
	backend := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		<-release // block until test cleanup
	}))
	defer backend.Close()
	defer close(release)

	cfg := &config.Config{
		Providers: []config.Provider{
			{Name: "test", BaseURL: backend.URL, Model: "deepseek-v4-pro", AuthStyle: "bearer"},
		},
		DefaultProvider: "test",
	}

	handler := NewHandler(cfg)
	// White-box: shrink the response-header timeout so the test is fast.
	handler.client = &http.Client{
		Transport: &http.Transport{
			ResponseHeaderTimeout: 200 * time.Millisecond,
		},
	}

	reqBody := `{"model":"deepseek-v4-pro","input":"Hello","stream":false}`
	req := httptest.NewRequest("POST", "/v1/responses", strings.NewReader(reqBody))
	req.Header.Set("Content-Type", "application/json")
	w := httptest.NewRecorder()

	handler.ServeHTTP(w, req)

	if w.Code != http.StatusBadGateway {
		t.Fatalf("status = %d, want %d (502) on backend header timeout", w.Code, http.StatusBadGateway)
	}
}
```

- [ ] **Step 3: Add the `client` field and build a staged-timeout client**

In `internal/proxy/handler.go`, update imports to include `"net"` and `"time"`, then change the struct and constructor:

```go
type Handler struct {
	cfg    *config.Config
	client *http.Client
}

func NewHandler(cfg *config.Config) *Handler {
	return &Handler{
		cfg: cfg,
		client: &http.Client{
			Transport: &http.Transport{
				DialContext:           (&net.Dialer{Timeout: 10 * time.Second}).DialContext,
				ResponseHeaderTimeout: 30 * time.Second,
				// No client.Timeout: body read (streaming) is intentionally unbounded.
			},
		},
	}
}
```

### Cycle A — convert reports scanner errors

- [ ] **Step 1: Write the failing test**

Add to `internal/convert/stream_test.go`. A single `data:` line larger than the default 64KB `bufio.Scanner` limit makes `scanner.Scan()` fail with `bufio.ErrTooLong`, exercising the `scanner.Err()` path. We assert an `error` event is emitted instead of the stream silently ending.

```go
func TestConvertStream_ScannerError(t *testing.T) {
	// One data line far larger than bufio's default 64KB token limit.
	huge := strings.Repeat("a", 100*1024)
	input := `data: {"id":"chatcmpl-err","choices":[{"index":0,"delta":{"content":"` + huge + `"}}]}`

	scanner := bufio.NewScanner(strings.NewReader(input)) // default 64KB buffer
	events := ConvertStream(scanner)

	var got []StreamEvent
	for ev := range events {
		got = append(got, ev)
	}

	if len(got) == 0 {
		t.Fatal("expected at least one event, got none")
	}
	last := got[len(got)-1]
	if last.Type != "error" {
		t.Fatalf("last event type = %q, want %q (scanner error must be surfaced, not swallowed)", last.Type, "error")
	}
}
```

- [ ] **Step 2: Run test to verify it fails**

Run: `go test -race ./internal/convert/ -run TestConvertStream_ScannerError -v`
Expected: FAIL — no `error` event today; the loop exits and the channel closes with the error swallowed (last event is whatever was emitted before, or none).

- [ ] **Step 3: Report `scanner.Err()` in `stream.go`**

In `internal/convert/stream.go`, inside the goroutine, immediately **after** the `for scanner.Scan() { ... }` loop and before the goroutine returns, add:

Note: `fmt` and the `escapeJSON` helper are already imported/defined in this file. `escapeJSON` returns a JSON-escaped string **without** surrounding quotes, so `%s` produces a valid JSON string value after the leading/closing quotes — verify the output parses (next step covers this implicitly via the test).

### Cycle B — handler raises the buffer so large lines stream intact

- [ ] **Step 6: Write the failing test**

Add to `internal/proxy/handler_test.go`. A streaming backend emits one SSE `data:` line whose content exceeds 64KB; we assert the proxy forwards the text delta intact (not truncated, no `error` event).

```go
func TestHandler_StreamingLargeLine(t *testing.T) {
	bigContent := strings.Repeat("x", 100*1024) // > 64KB single SSE line
	backend := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Content-Type", "text/event-stream")
		fmt.Fprintf(w, "data: {\"id\":\"chatcmpl-big\",\"choices\":[{\"index\":0,\"delta\":{\"role\":\"assistant\"}}]}\n")
		fmt.Fprintf(w, "data: {\"id\":\"chatcmpl-big\",\"choices\":[{\"index\":0,\"delta\":{\"content\":\"%s\"}}]}\n", bigContent)
		fmt.Fprintf(w, "data: {\"id\":\"chatcmpl-big\",\"choices\":[{\"index\":0,\"delta\":{},\"finish_reason\":\"stop\"}]}\n")
		fmt.Fprintf(w, "data: [DONE]\n")
		if f, ok := w.(http.Flusher); ok {
			f.Flush()
		}
	}))
	defer backend.Close()

	cfg := &config.Config{
		Providers: []config.Provider{
			{Name: "test", BaseURL: backend.URL, Model: "deepseek-v4-pro", AuthStyle: "bearer"},
		},
		DefaultProvider: "test",
	}
	handler := NewHandler(cfg)

	reqBody := `{"model":"deepseek-v4-pro","input":"Hi","stream":true}`
	req := httptest.NewRequest("POST", "/v1/responses", strings.NewReader(reqBody))
	req.Header.Set("Content-Type", "application/json")
	w := httptest.NewRecorder()

	handler.ServeHTTP(w, req)

	body := w.Body.String()
	if strings.Contains(body, "event: error") {
		t.Fatalf("stream produced an error event (line buffer too small):\n%s", body[:min(len(body), 300)])
	}
	if !strings.Contains(body, bigContent) {
		t.Fatal("large content was truncated or missing from the stream")
	}
}
```

`min` is a Go 1.21+ builtin (module is go 1.25) — no helper needed. Ensure `"fmt"` is in the test file's imports (add it if missing).

Expected: FAIL — the default 64KB scanner in `handleStreamingResponse` chokes on the >64KB line; the stream emits an `error` event (from Cycle A) and/or omits `bigContent`.
